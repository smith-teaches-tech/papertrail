Privacy Policy

Last updated: March 2026
The short version: PaperTrail does not collect, store, or transmit any personal data for Inspect or StyleMatch — everything runs locally in your browser. Verify transmits essay text to generate a report, but does not retain it. No student accounts. No advertising. No analytics.

What PaperTrail Accesses

PaperTrail reads revision history data from Google Docs using Google's built-in revision API — the same data that powers the native revision history feature inside Google Docs. This data includes document content at various points in time, edit timestamps, and anonymised author identifiers.

This data is accessed solely to display writing analysis results to the person using the extension. It is never transmitted to any server, stored beyond the current browser session, or shared with any third party.

StyleMatch — Text Never Leaves Your Browser

All StyleMatch analysis is performed locally on your device. The text you paste into the StyleMatch panel is never transmitted to PaperTrail Academic or any third party. There is no server-side component to StyleMatch — this is a design choice, not just a policy.

When you run a StyleMatch comparison, all eight stylometric calculations happen in your browser using JavaScript. Once you close the sidebar or navigate away, the text is cleared from memory.

Verify — Transmitted, Not Retained

Verify is the only PaperTrail tool that transmits text. When you run a Verify report, the submitted text is:

  • Sent to PaperTrail Academic's secure backend (Supabase, hosted on AWS infrastructure)
  • Forwarded to the Anthropic API for AI-assisted analysis
  • Returned as a structured report saved to your account

The full essay text is not retained by PaperTrail after the report is generated. Anthropic's standard API retention policy applies to their processing of submitted text — see Anthropic's data retention policy. Only report metadata is retained — scores, timestamps, and any label you assigned. Anthropic processes submitted text under terms that prohibit using it for model training. See anthropic.com/privacy for their full policy.

If your institution requires it, you should anonymize student-identifying information from text before submitting through Verify.

What PaperTrail Does Not Do

PaperTrail does not collect any personal information. It does not track usage, send analytics, display advertising, or sell data of any kind.

PaperTrail does not modify any Google Doc or leave any record of having been used on a document. It does not access student information systems, gradebooks, or any Google Workspace admin data.

Students never interact with PaperTrail. No student accounts are created. No student data is collected by PaperTrail Academic directly.

Chrome Extension Permissions

PaperTrail requests the following Chrome permissions in order to function:

activeTab and scripting — required to inject the revision analysis tool into the active Google Docs page.

tabs — required to open the playback window in a new tab and to detect when the user is on a Google Docs or Google Classroom page.

unlimitedStorage — required to temporarily hold revision data for large documents, which can contain tens of thousands of entries. This data is cleared when the session ends.

Host permissions for docs.google.com and classroom.google.com — required to operate on Google Docs and Google Classroom pages.

Google Classroom and Student Data

PaperTrail respects the access permissions that Google enforces on all documents. It can only read revision history that the logged-in teacher already has permission to view. It does not access any student data beyond what is visible in the document's standard revision history.

Because Inspect and StyleMatch process data locally without transmission, they do not raise additional data governance concerns beyond those of Google Docs itself. For Verify, see the section above.

Schools and IT coordinators with compliance questions should review our For Schools page, which covers FERPA, PIPEDA, and GDPR considerations in plain language.

Children's Privacy

PaperTrail does not knowingly collect any information from anyone, including minors. The extension is designed exclusively for use by educators. Document revision data accessed by Inspect is processed locally and discarded at the end of each session.

Changes to This Policy

If PaperTrail's data practices change — for example, if new features involve data transmission — this policy will be updated and the date above will reflect the change. Material changes will also be communicated to registered users by email.
